Refactor toàn bộ code JavaScript này để dễ đọc, dễ maintain và dễ review hơn.

Yêu cầu:
- Giữ nguyên logic và behavior hiện tại.
- Chuẩn hóa coding style trong toàn file.
- Tách logic phức tạp thành các function nhỏ, rõ responsibility.
- Đặt tên biến/function dễ hiểu và nhất quán.
- Ưu tiên early return để giảm nested code.
- Xóa duplicate code và xử lý dư thừa.
- Không over-engineering.

Quy tắc comment:
- Tất cả comment phải viết bằng tiếng Việt.
- Mỗi function cần có comment ngắn mô tả chức năng chính.
- Comment section/folder logic theo dạng:
  // --- Xử lý format dữ liệu ---
- Comment function theo dạng:
  /**
   * Chuẩn hóa dữ liệu người dùng trước khi gửi API
   */
- Chỉ comment phần mục đích hoặc business logic quan trọng.
- Không comment các dòng code hiển nhiên.
- Comment phải ngắn gọn, dễ scan khi review PR.

Coding style:
- Dùng async/await thống nhất nếu phù hợp.
- Tách condition phức tạp thành biến boolean có tên rõ nghĩa.
- Hạn chế side effects.
- Ưu tiên code flow dễ đọc.
